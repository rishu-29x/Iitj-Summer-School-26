# Q13. Arduino Pin Types

## 1. Digital Input
- Reads HIGH (1) or LOW (0) signals from external components.
- Voltage: 0V = LOW, 5V = HIGH.
- Example pin: D2, D3 (any digital pin set as INPUT)
- IoT use case: Reading a push button to trigger a smart door lock system.

## 2. Digital Output
- Sends HIGH or LOW signal to control external components.
- Example pin: D13 (built-in LED pin)
- IoT use case: Turning ON/OFF a relay to control a smart home appliance.

## 3. Analog Input
- Reads varying voltage levels (0V to 5V) and converts to values (0–1023) using ADC.
- Example pin: A0, A1, A2, A3, A4, A5
- IoT use case: Reading a soil moisture sensor to automate irrigation in a smart farm.

## 4. PWM Output (Pulse Width Modulation)
- Simulates analog output by sending pulses at varying duty cycles (0–255).
- Example pin: D3, D5, D6, D9, D10, D11 (marked with ~ on Arduino UNO)
- IoT use case: Controlling fan speed in a smart temperature regulation system.

## 5. I2C Pins (Inter-Integrated Circuit)
- Two-wire communication protocol: SDA (data) and SCL (clock).
- Example pin: A4 (SDA), A5 (SCL)
- IoT use case: Connecting multiple sensors (e.g. BME280 weather sensor) to one Arduino using just 2 wires.

## 6. SPI Pins (Serial Peripheral Interface)
- Four-wire high-speed communication: MOSI, MISO, SCK, SS.
- Example pin: D10 (SS), D11 (MOSI), D12 (MISO), D13 (SCK)
- IoT use case: Interfacing an SD card module to log sensor data in a remote weather station.
