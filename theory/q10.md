## Q10. What is a .gitignore File?

A `.gitignore` file tells Git which files and folders to **ignore** — Git will not track, stage, or commit them.
This keeps the repository clean by excluding compiled output, OS-generated files, temporary files, and IDE config folders that do not need to be shared.

### .gitignore for an Arduino/IoT Project

```
# Compiled output files (Arduino build artifacts)
*.hex
*.elf

# OS-specific files
.DS_Store
Thumbs.db

# IDE configuration folders
.vscode/
build/
```

### Why each rule is needed
- `*.hex` / `*.elf` — Compiled binary files generated when Arduino code is built. Always regenerated, no need to track.
- `.DS_Store` — Auto-created by macOS Finder. Not part of the project.
- `Thumbs.db` — Auto-created by Windows Explorer. Not part of the project.
- `.vscode/` — Personal VS Code editor settings, different for each developer.
- `build/` — Folder where Arduino IDE stores compiled files. Always regenerated.
