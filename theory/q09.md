Q9. Difference between git pull, git fetch, and git clone

git clone:
- Used ONCE to download a remote repository to your local machine for the first time.
- Creates a full copy of the repo including all files, branches, and commit history.
- Use when: You want to start working on a project that exists on GitHub but not on your computer.
- Example: git clone https://github.com/rishu-29x/iot-summer-school-2026.git

git fetch:
- Downloads new changes from the remote repo to your local machine, but does NOT merge them into your current branch.
- Your working files stay untouched. You can review changes before applying them.
- Use when: You want to see what others have pushed without affecting your own work yet.
- Example: git fetch origin

git pull:
- Downloads AND immediately merges the latest changes from the remote into your current branch.
- It is essentially git fetch + git merge in one command.
- Use when: You want to update your local branch with the latest changes from GitHub right away.
- Example: git pull origin main

Summary Table:
| Command    | Downloads | Merges | First Time Setup |
|------------|-----------|--------|-----------------|
| git clone  | Yes       | Yes    | Yes (once only) |
| git fetch  | Yes       | No     | No              |
| git pull   | Yes       | Yes    | No              |
